## Module 1: TNDS Brand Fundamentals

**Duration:** 30 minutes
**Goal:** Understand the TNDS brand identity and voice

### 1.1 Brand Identity

**Company:** True North Data Strategies
**Tagline:** Turning Data into Direction
**Positioning:** We fix operations for 5-20 person companies drowning in chaos

**Brand Colors:**
| Color | Hex | Use |
|-------|-----|-----|
| Navy | #1a3a5c | Primary headers, main text |
| Teal | #3d8eb9 | Secondary headers, highlights |
| Light Teal | #5ba3c8 | Backgrounds, subtle emphasis |

**Logo:** Compass rose design with "TRUE NORTH DATA STRATEGIES" and tagline

### 1.2 Voice and Tone

The TNDS voice is:
- **Direct** — Get to the point, no fluff
- **Confident** — We know our stuff, we don't need to prove it
- **Outcomes-focused** — What the client gets, not what we do
- **Owner-to-owner** — We speak as fellow operators

**The Core Rule:**
Write like you're explaining something to a smart business owner over coffee. No jargon. No corporate speak. Just clear, direct communication.

### 1.3 Voice Examples

**Wrong (Fix This):**
> "Our cutting-edge solution leverages AI-powered automation to optimize your operational workflows and enhance stakeholder alignment."

**Right:**
> "You'll see every job in real-time, so you know when problems happen—not three days later."

**Wrong:**
> "We strive to provide industry-leading customer service experiences."

**Right:**
> "You call, we answer. No phone trees. No chatbots."

### 1.4 Writing Rules

**DO:**
- Use contractions (don't, won't, can't)
- Write in active voice
- Use specific examples
- Quantify when possible
- Address the reader directly (you, your)
- Use short paragraphs (2-4 sentences)

**DON'T:**
- Use emojis (ever)
- Use buzzwords (synergy, leverage, ecosystem)
- Use jargon without defining it
- Hedge or apologize ("We try to...")
- Write long sentences (max 25 words)

### 1.5 Knowledge Check

Answer these questions:

1. What are the two main brand colors?
2. Which voice is correct: "Our solution leverages..." or "You'll see..."?
3. Can you use emojis in TNDS documents?
4. What's the maximum recommended sentence length?

**Answers:**
1. Navy (#1a3a5c) and Teal (#3d8eb9)
2. "You'll see..." (outcomes-focused)
3. No, never
4. 25 words

## Module 2: Document Types

**Duration:** 45 minutes
**Goal:** Know when to use each document type and template

### 2.1 Document Type Overview

| Type | Purpose | Format | Audience |
|------|---------|--------|----------|
| Client-Facing Service | Describe services | Word | Clients |
| Proposal | Present project scope | Word | Clients |
| Agreement | Formalize engagement | Word | Clients |
| SOP | Standardize processes | Markdown | Team |
| Diagnostic/Cheat Sheet | Quick reference tool | Word | Sales/Team |
| Module Documentation | Document Command modules | Either | Technical |

### 2.2 When to Use Each Type

**Client-Facing Service Document**
- Describing a service offering
- Explaining what clients get
- Marketing collateral

**Proposal**
- After MAP session
- Presenting recommended scope
- Requesting commitment

**Agreement**
- Client has accepted proposal
- Formalizing the engagement
- Legal protection

**SOP (Standard Operating Procedure)**
- Process needs to be repeatable
- Multiple people do the same task
- Compliance or consistency required

**Diagnostic/Cheat Sheet**
- Quick reference during calls
- Assessment tool with scripts
- Decision support tool

**Module Documentation**
- Documenting Command modules
- Technical implementation details
- Client-facing module benefits

### 2.3 Template Selection Decision Tree

```
Is this for a client?
├── YES → What are we asking them to do?
│         ├── Understand our service → Service Document
│         ├── Approve project scope → Proposal
│         └── Sign agreement → Agreement
│
└── NO → What's the purpose?
          ├── Standardize a process → SOP
          ├── Quick reference tool → Cheat Sheet
          └── Document a module → Module Doc
```

### 2.4 Key Sections by Template

**Service Document:**
- What This Engagement Is
- Who This Is For
- Problems This Removes
- What You Get
- Scope & Pricing

**Proposal:**
- Executive Summary
- Current Situation
- Proposed Solution
- Scope & Deliverables
- Investment & Payment

**SOP:**
- Purpose
- Scope
- Roles & Responsibilities
- Procedure Steps
- Quality Checks

### 2.5 Knowledge Check

Match the situation to the correct document type:

1. Client wants to know what Command Center Build includes
2. MAP session is done, need to present recommendations
3. Client said yes, need to formalize terms
4. Team needs consistent process for onboarding clients
5. Sales team needs a reference for ASSESS calls

**Answers:**
1. Service Document
2. Proposal
3. Agreement
4. SOP
5. Cheat Sheet

## Module 3: The Documentation Workflow

**Duration:** 30 minutes
**Goal:** Follow the standard process for creating documents

### 3.1 The Five Phases

```
INTAKE → DEVELOPMENT → QA → FINALIZATION → DELIVERY
```

### 3.2 Phase 1: Intake (5-10 min)

**Step 1: Clarify the Request**
Ask:
- Who will read this?
- What action should they take after?
- Word or Markdown?
- When is it needed?

**Step 2: Select Template**
Use the decision tree from Module 2.

**Step 3: Gather Information**
Collect everything you need before writing.

### 3.3 Phase 2: Development (15-60 min)

**Step 1: Structure Content**
Copy the template structure. Plan what goes in each section.

**Step 2: Write Content**
- Draft section by section
- Apply TNDS voice
- Stay specific

**Step 3: Format Document**
Apply brand formatting (colors, fonts, spacing).

### 3.4 Phase 3: Quality Assurance (5-10 min)

Use the QUALITY-REVIEW-CHECKLIST.md:
- Voice check (direct, outcomes-focused)
- Brand check (colors, fonts, logo)
- Structure check (template followed)

### 3.5 Phase 4: Finalization (5 min)

- Final formatting pass
- Name file correctly: `TNDS_[Type]_[Name]_[Date].ext`
- Save to correct location

### 3.6 Phase 5: Delivery

- Share document appropriately
- Confirm receipt
- Handle feedback

### 3.7 Knowledge Check

Put these steps in order:
- [ ] Format the document
- [ ] Save with correct file name
- [ ] Clarify the request
- [ ] Quality review
- [ ] Write content
- [ ] Select template

**Correct Order:**
1. Clarify the request
2. Select template
3. Write content
4. Format the document
5. Quality review
6. Save with correct file name

## Module 4: Integration with Other Skills

**Duration:** 30 minutes
**Goal:** Understand when Documentation is triggered by other skills

### 4.1 The Documentation Connection

Documentation is the "output layer" for all TNDS skills. Other skills analyze and guide; Documentation produces deliverables.

```
Other Skills (Analysis) → Documentation Skill (Output)
```

### 4.2 Direction Protocol Integration

| Stage | Triggers | Document |
|-------|----------|----------|
| ASSESS | Call completed | ASSESS summary |
| MAP | Session completed | MAP summary, Gap report |
| CHART | Findings ready | Proposal |
| LAUNCH | Client accepts | Agreement |

### 4.3 Command Protocol Integration

| Phase | Triggers | Document |
|-------|----------|----------|
| Kickoff | Project starting | Kickoff doc |
| Development | Module built | Module documentation |
| Training | Ready for training | Training materials |
| Handoff | Project complete | Handoff document |

### 4.4 Other Skill Integrations

**World Model Mapper:**
- Process mapped → Process documentation
- Gaps identified → Gap report

**Bearing Check:**
- Decision made → Decision record
- Analysis complete → Decision log entry

### 4.5 Trigger Phrases

When you hear these, invoke Documentation:
- "Create a proposal for..."
- "Document this session..."
- "Create an SOP for..."
- "Turn this into a document..."

### 4.6 Knowledge Check

What document should be created when:

1. Direction Protocol MAP session is complete
2. Command Protocol module is built
3. Bearing Check decision is made
4. World Model Mapper identifies gaps

**Answers:**
1. MAP summary and/or Proposal
2. Module documentation
3. Decision record
4. Gap report

## Practice Exercises

**Duration:** 60-90 minutes total
**Goal:** Create actual documents using what you've learned

### Exercise 1: Service Document (20 min)

**Task:** Create a service document for "Battle Rhythm Install"

**Information:**
- Price: $2,000-$4,000
- Timeline: 2-4 weeks
- Target: Operations 5-15 people, struggling with consistency
- Problems: Different person does it differently, owner has to manage everything
- Deliverables: Operating cadence, accountability systems, meeting rhythms

**Steps:**
1. Use CLIENT-FACING SERVICE template
2. Write all required sections
3. Apply TNDS voice
4. Format as Word document

**Self-Check:**
- [ ] Used outcomes language
- [ ] Pain points in client voice
- [ ] Specific deliverables listed
- [ ] Brand formatting applied

---

### Exercise 2: SOP (25 min)

**Task:** Create an SOP for "Responding to New Prospect Inquiries"

**Information:**
- Trigger: New inquiry received (email, form, phone)
- Who: Sales team
- Steps: Log in CRM, acknowledge within 4 hours, qualify, schedule IDENTIFY call
- Quality: Must log all contacts, must respond within 4 hours

**Steps:**
1. Use SOP template
2. Write all required sections
3. Include quality checks
4. Format as Markdown

**Self-Check:**
- [ ] Each step has Who, What, When, How
- [ ] Quality checks included
- [ ] Exceptions addressed
- [ ] Version info in header

---

### Exercise 3: Cheat Sheet (20 min)

**Task:** Create a cheat sheet for "IDENTIFY Call"

**Information:**
- Duration: 15-20 minutes
- Purpose: Determine if prospect is a fit
- Key questions: What's broken? How long? Who decides? Budget reality?
- Red flags: No budget authority, wants silver bullet, disrespectful
- Green flags: Operations owner, ready to invest, specific pain

**Steps:**
1. Use DIAGNOSTIC template
2. Write exact scripts
3. Include red/green flags
4. Format as Word document

**Self-Check:**
- [ ] Scripts are exact words
- [ ] Timing included
- [ ] Red flags defined
- [ ] Action for flag count specified

---

### Exercise 4: Proposal Section (15 min)

**Task:** Write an Executive Summary for a proposal

**Context:**
- Client: Peak HVAC (10 technicians)
- MAP Findings: No visibility into job status, invoicing takes 3 days, can't take a day off
- Proposed Solution: Command Center Build with data-command and financial-command modules
- Investment: $4,000

**Steps:**
1. Write 3-4 paragraphs
2. Summarize situation, solution, investment
3. Use outcomes language
4. Keep to 1 page maximum

**Self-Check:**
- [ ] Situation clearly stated
- [ ] Solution tied to problems
- [ ] Investment mentioned
- [ ] Outcomes-focused language
