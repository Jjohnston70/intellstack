# Documentation Email Templates

**Version:** 1.0
**Last Updated:** January 29, 2026
**Purpose:** Standard email templates for document-related communications

---

## Overview

These templates ensure consistent, professional communication when delivering, requesting, or following up on TNDS documents.

**Voice Guidelines:**
- Direct and clear
- No excessive pleasantries
- Action-oriented
- Professional but not stiff

---

## Part 1: Document Delivery Emails

**Subject:** Your Command Center Proposal – [Client Name]

- Timeline: [X] weeks
- Payment: [40/30/30 or 50/50]

The proposal covers what we found, what we recommend, and what you can expect.

Take a look and let me know what questions come up. I'm available [day/time] if you want to walk through it together.

Jacob Johnston

## Part 2: Document Request Clarification Emails

### Template 2A: Initial Clarification Request

**Use When:** Need more information before creating document

**Subject:** Re: [Original Request] – Quick Questions

---

[First Name],

Before I create [document type], I need a few things:

1. [Question 1]?
2. [Question 2]?
3. [Question 3]?

Quick answers are fine. I'll start once I have these.

Jacob

### Template 2B: Follow-Up for Missing Information

**Use When:** Already asked, haven't received answer

**Subject:** Re: [Original Request] – Following Up

---

- [Missing item 1]
- [Missing item 2]

Let me know if you want to hop on a quick call instead of email.

Jacob

## Part 5: Internal Document Communication

**Use When:** Rolling out new standard operating procedure

**Subject:** New SOP: [Process Name] – Effective [Date]

---

Team,

We have a new SOP for [Process Name].

**Why:** [Brief reason—consistency, efficiency, etc.]

**What Changes:**
- [Change 1]
- [Change 2]

**Effective:** [Date]

Questions? Reply to this email or bring to [meeting/standup].

Jacob

## Part 6: Quick Reference

### Email Type Selection

| Situation | Template |
|-----------|----------|
| Sending proposal | 1A |
| Sending agreement | 1B |
| Sending service doc | 1C |
| Sharing internal SOP | 1D |
| Sending training materials | 1E |
| Need info before creating | 2A |
| Following up for info | 2B |
| Acknowledging revision request | 3A |
| Delivering revised document | 3B |
| Clarifying revision request | 3C |
| Checking if received | 4A |
| Following up on proposal | 4B |
| Following up on agreement | 4C |
| Requesting feedback | 4D |
| Announcing new SOP | 5A |
| Updating existing SOP | 5B |
| Requesting internal review | 5C |

### Subject Line Formulas

| Type | Formula | Example |
|------|---------|---------|
| Delivery | [Document Type] – [Client/Topic] | Your Command Center Proposal – Peak HVAC |
| Follow-up | Re: [Document] – [Question/Action] | Re: Service Agreement – Next Steps |
| Internal | [Action]: [Process Name] | New SOP: Client Onboarding |
| Revision | Updated: [Document Name] | Updated: Peak HVAC Proposal |

### Timing Guidelines

| Email Type | Send When |
|------------|-----------|
| Proposal follow-up | 3-5 business days after sending |
| Agreement follow-up | 2-3 business days after sending |
| Receipt check | 2 business days if no response |
| Revision delivery | Same day or next day |
