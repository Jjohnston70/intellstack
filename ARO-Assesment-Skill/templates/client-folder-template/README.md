## How to Use This Folder

This folder contains all files for the ARO Assessment engagement. Copy this entire template to `20_PROSPECTS/[Client-Name]-ARO-Assessment/` when starting a new assessment.

### File Purposes

| File | Purpose | When to Update |
|------|---------|----------------|
| `intake-questionnaire-responses.md` | Client's answers to intake questions | When received from client |
| `discovery-notes.md` | Notes from 90-min discovery call | During/after discovery call |
| `scoring-worksheet.md` | 6-area scoring with justifications | Day 3-4 (analysis phase) |
| `context-architecture-report.md` | Main deliverable for client | Day 5-6 (generated) |
| `implementation-roadmap.md` | Quick wins + phased plan | Day 5-6 (generated) |
| `context-graph.md` | AI operational reference | Day 5-6 (generated) |

### Workflow

1. **Save intake responses** → `intake-questionnaire-responses.md`
2. **Prepare for discovery** → Review intake, prepare questions
3. **Conduct discovery call** → Take notes in `discovery-notes.md`
4. **Score operations** → Complete `scoring-worksheet.md`
5. **Generate deliverables** → Create remaining files
6. **Quality review** → Self-check + peer review
7. **Deliver to client** → Send all 4 deliverables
8. **Handoff call** → Present findings, get decision

### After Handoff

Based on client decision, this folder moves:

| Decision | Folder Moves To |
|----------|-----------------|
| YES (Implementation) | `30_ACTIVE_CLIENTS/[Client]/00-ARO-Assessment/` |
| DIY | `40_PAST_CLIENTS/[Client]-ARO-DIY/` |
| Not Now | `25_NURTURE/[Client]-ARO-Pending/` |
| Goes Dark | `90_ARCHIVE/[Year]/[Client]-ARO-NoResponse/` |

See `FOLDER-STRUCTURE.md` for complete folder lifecycle documentation.
