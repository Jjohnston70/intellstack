## Part 1: Service Positioning

### ARO Assessment (Direction Protocol - ASSESS Stage)

**What It Is:** A paid diagnostic that evaluates a business's operational readiness for AI automation.

| Attribute | Detail |
|-----------|--------|
| **Protocol** | Direction Protocol (Sales) |
| **Stage** | ASSESS |
| **Duration** | 7 days |
| **Investment** | $1,500 - $2,500 |
| **Deliverables** | Context Architecture Report, Implementation Roadmap, Context Graph v1.0, Documentation Tier Recommendation |
| **Output** | Score (1-5) + Prioritized friction points + Service recommendation |

**Key Point:** The ARO Assessment is **complete in itself**. Clients receive actionable deliverables they can use for DIY implementation OR as the foundation for TNDS-led implementation.

---

### Command Protocol Services (Implementation)

Command Protocol is TNDS's 12-phase delivery methodology. Three productized services use this protocol:

#### Service 1: Command Center Build

**What It Is:** One-time project to build the operational foundation.

| Attribute | Detail |
|-----------|--------|
| **Protocol** | Command Protocol (Phases 1-8 minimum) |
| **Duration** | 4-8 weeks |
| **Investment** | $5,000 - $8,000 |
| **Scope** | Core systems setup, integrations, documentation |
| **Output** | Working Command Center + Documentation + Training |
| **Support** | 30-day warranty period |

**Best For:**
- Businesses with internal capacity to maintain systems
- Clear, defined scope
- Budget-conscious clients
- ARO Scores 2.5-4.0

**What's Included:**
- Context Graph v2.0 (upgraded from assessment)
- Core system configuration
- Key integrations built
- Documentation (Memory-Bank or SpecKit per tier)
- Training session
- 30-day support

---

#### Service 2: Battle Rhythm

**What It Is:** Command Center Build + ongoing maintenance partnership.

| Attribute | Detail |
|-----------|--------|
| **Protocol** | Command Protocol (All 12 phases) |
| **Duration** | Build: 4-8 weeks, Maintenance: Monthly retainer |
| **Investment** | Build: $5,000-$8,000 + Retainer: $500-$1,500/month |
| **Scope** | Full build + monthly optimization + support |
| **Output** | Maintained Command Center + Continuous improvement |
| **Support** | Ongoing |

**Best For:**
- Businesses without technical staff
- Clients wanting "set and forget" operations
- Growing businesses needing ongoing optimization
- ARO Scores 2.0-3.5

**What's Included:**
- Everything in Command Center Build
- Context Graph v3.0 (maintained)
- Monthly optimization reviews
- Proactive system monitoring
- Priority support
- Quarterly business reviews

---

#### Service 3: Command Partner

**What It Is:** Fractional operations partnership — TNDS as embedded team member.

| Attribute | Detail |
|-----------|--------|
| **Protocol** | Command Protocol (All phases, recurring) |
| **Duration** | Ongoing partnership |
| **Investment** | $2,000 - $5,000/month |
| **Scope** | Full operational ownership of defined systems |
| **Output** | Managed operations + Strategic guidance |
| **Support** | Dedicated partner relationship |

**Best For:**
- Businesses needing operational leadership, not just tools
- Complex multi-system environments
- High-growth companies
- Regulated industries (Tier 3 documentation)
- ARO Scores below 2.5 OR highly complex operations

**What's Included:**
- Everything in Battle Rhythm
- Strategic operations planning
- Vendor/tool management
- Team training and change management
- Executive reporting
- Expansion planning

## Part 2: Score-Based Recommendations

### The ARO Score

The ARO Assessment produces a composite score (1.0-5.0) across 6 operational areas:

| Score | Label | Meaning |
|-------|-------|---------|
| 1.0-1.9 | **Chaotic** | No structure, tribal knowledge only |
| 2.0-2.9 | **Poor** | Some attempts but inconsistent |
| 3.0-3.9 | **Functional** | Works but with friction |
| 4.0-4.9 | **Good** | Mostly organized, minor gaps |
| 5.0 | **Agent-Ready** | Explicit, consistent, accessible |

### Decision Factors Beyond Score

The ARO Score is the primary indicator, but these factors can shift recommendations:

#### Factor 1: Client Capacity

| Question | If YES | If NO |
|----------|--------|-------|
| Do they have someone technical on staff? | Can self-maintain → Build | Needs ongoing support → Rhythm/Partner |
| Do they have time for training? | Build + training works | Partner may be better |
| Have they successfully managed similar projects? | Higher confidence in Build | Consider Rhythm |

#### Factor 2: Complexity

| Question | If YES | If NO |
|----------|--------|-------|
| Multiple business lines or locations? | Consider Partner | Build or Rhythm |
| Compliance/regulatory requirements? | Tier 3 + Partner | Standard tiers |
| 5+ systems to integrate? | Consider Partner | Build can handle |

#### Factor 3: Budget

| Budget Range | Viable Options |
|--------------|----------------|
| < $5,000 total | DIY with assessment deliverables |
| $5,000-$8,000 one-time | Command Center Build |
| $5K-$8K + $500-$1.5K/mo | Battle Rhythm |
| $2K-$5K/mo ongoing | Command Partner |

#### Factor 4: Urgency

| Timeline | Recommendation |
|----------|----------------|
| Need results in < 30 days | Quick Wins from assessment + future Build |
| 30-90 days available | Command Center Build |
| Ongoing improvement mindset | Battle Rhythm or Partner |

## Part 4: Handoff Conversation Guide

### Presenting the Recommendation

After presenting the assessment findings, use this framework to present the service recommendation:

#### Script Template

> "Based on your ARO score of [X.X] and what we discussed about your [capacity/budget/timeline], I recommend **[Service Name]**."
>
> "Here's why this is the right fit:"
> - **[Reason 1 tied to their specific situation]**
> - **[Reason 2 tied to their specific pain point]**
> - **[Reason 3 tied to their goals]**
>
> "The investment for [Service] is **[$X,XXX]** [one-time / plus $X/month]."
>
> "This includes [list 3-4 key deliverables]."
>
> "Remember, your $500 assessment credit applies toward this, bringing your initial investment to **[$X,XXX]**."

### Handling Common Objections

#### "That's more than I expected"

> "I understand. Let me break down what's included and why it's priced this way..."
>
> [Walk through scope and deliverables]
>
> "We could also start with [smaller scope] at [$lower amount] and add [remaining scope] later. Would that work better for your budget?"

#### "Can I just do it myself?"

> "Absolutely — that's what the assessment deliverables are for. You have everything you need to implement the Quick Wins and Phase 1 yourself."
>
> "Most DIY clients find they can handle 60-70% of the recommendations. The question is whether the 30-40% that's harder is worth the time investment."
>
> "You have 30 days of email support from me either way. Try the Quick Wins first, and if you hit a wall, we can talk about having us take over specific pieces."

#### "I need to think about it"

> "Of course. What specifically would you like to think through? Is it budget, timing, or something about the scope?"
>
> [Address the specific concern]
>
> "The assessment credit is good for 90 days, so there's no rush. When would be a good time to reconnect?"

## Part 5: Post-Decision Handoffs

### If Client Chooses TNDS Implementation

**Immediate Actions:**
1. Send implementation proposal (EMAIL-TEMPLATES #5)
2. Update CRM to Stage 4A: Proposal Sent
3. Update deal value to implementation amount
4. Schedule follow-up for 3 days

**Upon Signed Agreement:**
1. Move folder: `20_PROSPECTS/` → `30_ACTIVE_CLIENTS/`
2. Upgrade Context Graph: v1.0 → v2.0
3. Create implementation folder structure (per FOLDER-STRUCTURE.md)
4. Schedule implementation kickoff
5. Begin Command Protocol Phase 1: Recon - Discovery

### Command Protocol Kickoff Checklist

When transitioning from ARO Assessment to Command Protocol:

- [ ] Context Graph v2.0 updated with implementation details
- [ ] Assessment folder moved to `00-ARO-Assessment/` within client folder
- [ ] Implementation folder structure created
- [ ] Kickoff call scheduled
- [ ] First payment received (50% or 40%)
- [ ] Documentation tier confirmed (Tier 1, 2, or 3)
- [ ] Memory-Bank initialized (if Tier 2+)
- [ ] SpecKit initialized (if Tier 2+)
