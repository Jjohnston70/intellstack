## The Six Core Skills

### 1. Bearing Check - Decision Validation Framework

**What It Does:**
Bearing Check is an eight-checkpoint system that pressure-tests any significant decision before you commit resources. It guards against the seven most common cognitive biases that derail business decisions.

**The Problem It Solves:**
People make bad decisions not because they lack information, but because they skip critical thinking steps. Bearing Check forces systematic validation before action.

**The Eight Checkpoints:**

| Checkpoint | Question It Answers |
|------------|---------------------|
| Ground Truth | What's the actual success rate for decisions like this? |
| Graveyard Recon | Why have others failed at this exact thing? |
| Mechanism Check | Specifically, *why* would this work? |
| Fixed Points | What durable advantages do you have? |
| Odds Assessment | What's your realistic probability of success? |
| Constraint Fit | Do you actually have what's needed to execute? |
| Staged Advance | Can you test this small before going big? |
| Pre-Mortem | If this fails, what's the most likely cause? |

**Cognitive Traps It Guards Against:**
- Survivorship bias (only seeing winners)
- Confirmation bias (seeking agreeable information)
- Sunk cost fallacy (honoring past investments over future value)
- Overconfidence bias (inflated self-assessment)
- Planning fallacy (underestimating time and resources)
- Narrative fallacy (creating false stories from random events)
- Status quo bias (preferring inaction over action)

**When To Use It:**
- Before accepting a new client
- Before launching a new service offering
- Before making significant investments
- Before career changes
- Before strategic pivots
- Anytime stakes are high and reversal is costly

**Output:** A GO/NO-GO recommendation with documented reasoning, risk factors, and conditions for success.

### 2. ARO Assessment - Agent-Ready Operations Evaluation

**What It Does:**
ARO (Agent-Ready Operations) Assessment evaluates how prepared a business operation is for AI automation and augmentation. It produces a scored report with a visual "Context Graph" showing where AI can add value.

**The Problem It Solves:**
Most businesses know AI could help them but don't know *where* or *how*. ARO Assessment provides a structured methodology for identifying AI opportunities and readiness gaps.

**The Five Assessment Dimensions:**

| Dimension | What It Measures | Score Range |
|-----------|------------------|-------------|
| Data Accessibility | Can systems access the information they need? | 1-5 |
| Process Documentation | Are procedures written down and consistent? | 1-5 |
| Decision Clarity | Are decision rules explicit or tribal knowledge? | 1-5 |
| Integration Readiness | Can systems talk to each other? | 1-5 |
| Human-AI Handoff | Are escalation and oversight paths clear? | 1-5 |

**Deliverables:**
- Context Graph visualization
- Dimension-by-dimension scoring
- Prioritized opportunity list
- Implementation roadmap with phases
- Risk and dependency mapping

**When To Use It:**
- Prospect is interested in "AI for their business"
- Client wants to automate but doesn't know where to start
- Pre-engagement scoping for automation projects
- Technology modernization planning

### 3. World Model Mapper - Process Reality Analysis

**What It Does:**
World Model Mapper creates a comprehensive map of how an operation actually works versus how people *think* it works. It exposes shadow processes, hidden dependencies, and the true "physics" that constrain what's possible.

**The Problem It Solves:**
Most process documentation describes the ideal, not the real. Decisions made on inaccurate process understanding fail. World Model Mapper surfaces the truth.

**The Five Mapping Phases:**

| Phase | What Gets Mapped |
|-------|------------------|
| STATE | What exists - people, data, systems, entities |
| ACTIONS | What can be done - operations, decisions, transformations |
| TRANSITIONS | What causes change - triggers, events, state changes |
| FEEDBACK | What signals exist - responses, metrics, alerts |
| PHYSICS | What can't change - regulations, laws, platform limits |

**Key Concepts:**

- **Shadow Processes:** The unofficial workarounds that actually run the business
- **Domain Physics:** The immutable constraints that no amount of effort can change
- **Feedback Loops:** The cycles that amplify or dampen outcomes over time
- **Reality vs. Perception Gap:** The distance between documented and actual operations

**Output:** A Gap Report documenting the true operational state, shadow processes, constraint inventory, and recommendations.

### 4. Direction Protocol - Client Acquisition System

**What It Does:**
Direction Protocol is a five-stage methodology for qualifying prospects, understanding their needs, and closing deals. It provides scripts, timing guidance, and qualification criteria for each stage.

**The Problem It Solves:**
Most salespeople wing it. They don't know when to qualify out, when to propose, or how to price. Direction Protocol removes guesswork from client acquisition.

**The Five Stages:**

| Stage | Duration | Objective | Key Question |
|-------|----------|-----------|--------------|
| IDENTIFY | 15-20 min | Determine fit | Is there a problem worth solving? |
| ASSESS | 30 min | Validate problem | Is this problem real and painful enough? |
| MAP | 90-120 min | Understand scope | What's broken and what's needed? |
| CHART | 30-45 min | Present solution | Here's what we'll do and what it costs |
| LAUNCH | Variable | Close deal | Sign agreement, collect deposit, begin |

**Qualification Scoring:**
Each prospect is scored 1-3 on five factors:
- Number of employees
- Current organizational state
- Data source complexity
- Owner availability
- Technical comfort level

**Pricing Tiers:**
- 5-7 points = Standard engagement
- 8-11 points = Professional engagement
- 12-15 points = Enterprise engagement

**Integrations:**
- ARO Assessment can be offered during MAP for AI-focused prospects
- World Model Mapper can be used during MAP for complex operations
- Bearing Check validates major pursuit decisions

### 5. Command Protocol - Service Delivery System

**What It Does:**
Command Protocol is a twelve-phase framework for delivering client projects from kickoff to ongoing partnership. It provides time estimates, checklists, templates, and quality gates for each phase.

**The Problem It Solves:**
Service delivery often becomes chaotic and inconsistent. Command Protocol ensures every client receives the same high-quality, systematic experience.

**The Twelve Phases:**

| Phase | Name | Time Est. | Key Activities |
|-------|------|-----------|----------------|
| 1 | Post-Launch Setup | 1 hr | Admin setup, folder structure |
| 2 | Kickoff | 1 hr | Scope confirmation, success metrics |
| 3 | Technical Setup | 2 hrs | Access configuration, environment prep |
| 4 | Development | 4 hrs | Core system build using modules |
| 5 | Customization | 2 hrs | Branding, business logic application |
| 6 | OAuth & Auth | 0.5 hr | Client authorization |
| 7 | Testing | 1 hr | Validation, bug fixing |
| 8 | Documentation | 1 hr | User guides, reference materials |
| 9 | Training & Launch | 1 hr | Client training, go-live |
| 10 | Post-Launch Support | 1.5 hrs | Monitoring, adjustments |
| 11 | Handoff to Support | As needed | Support transition |
| 12 | Ongoing | Continuous | Command Partner relationship |

**Total Standard Engagement:** ~15 hours

**Module Library:**
Command Protocol references a library of pre-built modules that can be installed:
- data-command, financial-command, analytics-command
- asset-command, office-command, onboard-command
- proposal-command, workspace-command, sms-command
- contract-command, realty-command, tax-command
- seo-command, govcon-command, compliance-gov-module

## The Philosophy Behind Command-Post

Command-Post is built on five core principles:

**1. Reality Over Perception**
Document what actually happens, not what should happen. Shadow processes are features, not bugs - they reveal where official processes fail.

**2. Systematic Over Intuitive**
Replace gut feelings with checklists and frameworks. Good judgment is built, not born. Systems catch what intuition misses.

**3. Integrated Over Isolated**
Skills that work together multiply their value. A decision framework that feeds a sales process that feeds a delivery system creates compounding benefits.

**4. Documented Over Remembered**
If it's not written down, it doesn't exist. Every decision, every process, every lesson learned gets captured for future reference.

**5. Honest Over Comfortable**
Bearing Check asks hard questions. World Model Mapper reveals inconvenient truths. Reality-based decisions outperform optimism-based decisions.
