# Direction Protocol Quality Review Checklist

**Version:** 1.0
**Last Updated:** January 28, 2026
**Purpose:** Ensure consistent quality across all Direction Protocol stages

---

## Overview

Quality review happens at key transition points in Direction Protocol:
1. After IDENTIFY → Before scheduling ASSESS
2. After ASSESS → Before scheduling MAP
3. After MAP → Before CHART presentation
4. After CHART → Before sending agreement

---

## Stage 1: IDENTIFY Quality Review

### When to Review
Before scheduling ASSESS call

### Reviewer
Self-review (quick check)

### Checklist

**Qualification Completeness:**
- [ ] Three core questions asked
- [ ] Qualification score calculated (0-8)
- [ ] Pain points documented with specifics
- [ ] Disqualifiers checked
- [ ] Decision-maker status confirmed

**Documentation:**
- [ ] IDENTIFY-notes.md created
- [ ] Contact information complete
- [ ] Lead source documented
- [ ] Key quotes captured

**CRM:**
- [ ] Deal created
- [ ] Stage updated
- [ ] Next action set
- [ ] Expected value estimated

**Quality Standards:**
- [ ] Score justification makes sense (not just numbers)
- [ ] At least one specific pain point documented
- [ ] No obvious red flags ignored

### Pass Criteria
- Score 5+ to proceed to ASSESS
- All required documentation complete
- No unaddressed red flags

---

## Stage 2: ASSESS Quality Review

### When to Review
Before scheduling MAP session

### Reviewer
Self-review (quick check)

### Checklist

**Discovery Completeness:**
- [ ] All five sections covered
- [ ] SRO score calculated (0-12)
- [ ] Pain points quantified where possible
- [ ] Current state documented
- [ ] Budget discussed

**Documentation:**
- [ ] ASSESS-notes.md created
- [ ] Operational reality captured
- [ ] Stakes documented
- [ ] Readiness evaluated
- [ ] Openness assessed

**CRM:**
- [ ] Stage updated
- [ ] SRO score recorded
- [ ] MAP date set
- [ ] Deal value updated if needed

**Quality Standards:**
- [ ] Pain is specific, not generic
- [ ] Cost/impact quantified (time or money)
- [ ] Readiness assessment realistic
- [ ] Concerns documented (if any)

### Pass Criteria
- SRO score 6+ to proceed to MAP
- Specific pain with quantified impact
- Budget conversation had
- Client committed to MAP time investment

---

## Stage 3: MAP Quality Review

### When to Review
Before CHART presentation

### Reviewer
Peer review recommended (if available)

### Checklist

**12 Diagnostic Questions:**
- [ ] Q1: Business description captured
- [ ] Q2: Revenue activities documented
- [ ] Q3: Team/roles mapped
- [ ] Q4: Job flow documented
- [ ] Q5: Breakdown points identified
- [ ] Q6: Visibility mechanisms noted
- [ ] Q7: Recurring decisions listed
- [ ] Q8: Friction points identified
- [ ] Q9: Information gaps noted
- [ ] Q10: Owner dependency assessed
- [ ] Q11: Knowledge distribution mapped
- [ ] Q12: Undocumented rules captured

**Complexity Scoring:**
- [ ] All 5 factors scored
- [ ] Scores justified with specifics
- [ ] Total calculated correctly
- [ ] Base price identified
- [ ] Adjustments applied correctly
- [ ] Final price documented

**Solution Design:**
- [ ] Command modules identified
- [ ] Module selection justified
- [ ] Timeline estimated
- [ ] Client involvement defined
- [ ] Constraints documented

**Context Graph:**
- [ ] v1.0 created
- [ ] Entity profile complete
- [ ] Business context populated
- [ ] Pain points documented
- [ ] Tech stack captured

**CHART Materials:**
- [ ] Current state summary ready
- [ ] Data flow diagram created
- [ ] Findings list prepared
- [ ] Pricing breakdown ready
- [ ] Proposal draft complete

**CRM:**
- [ ] Stage updated
- [ ] Complexity score recorded
- [ ] Proposed price updated
- [ ] CHART date set

### Quality Standards

**Scoring Quality:**
| Check | Pass | Fail |
|-------|------|------|
| Scores match findings | Justification clear | Numbers don't match narrative |
| Price calculation correct | Math checks out | Calculation errors |
| Modules appropriate | Tied to pain points | Generic recommendations |
| Timeline realistic | Based on complexity | Arbitrary |

**Documentation Quality:**
| Check | Pass | Fail |
|-------|------|------|
| Specific examples | Real situations from client | Generic descriptions |
| Quantified impact | Numbers attached | Vague statements |
| Client voice | Their words used | Our jargon |
| Actionable findings | Clear what to fix | Observation without solution |

### Pass Criteria
- All 12 questions answered with specifics
- Scoring complete and justified
- Price verified (math correct)
- All CHART materials ready
- Context Graph v1.0 complete

## Stage 4: CHART Quality Review

### When to Review
Before sending service agreement

### Reviewer
Self-review (quick check)

### Checklist

**Presentation Delivery:**
- [ ] Current state validated by client
- [ ] Findings presented clearly
- [ ] Recommendation explained
- [ ] Pricing stated confidently
- [ ] Outcome painted vividly
- [ ] Questions answered

**Client Response:**
- [ ] Decision captured (Yes/No/Not Now)
- [ ] Objections documented
- [ ] Concerns addressed
- [ ] Next steps agreed

**Proposal/Agreement:**
- [ ] Scope matches what was discussed
- [ ] Price matches verbal quote
- [ ] Timeline accurate
- [ ] Payment terms clear
- [ ] Deliverables listed
- [ ] No errors in client name/details

**CRM:**
- [ ] Stage updated appropriately
- [ ] Proposal sent date recorded
- [ ] Follow-up schedule set
- [ ] Notes complete

### Quality Standards

**Agreement Quality:**
| Check | Pass | Fail |
|-------|------|------|
| Scope clarity | Specific deliverables | Vague language |
| Price accuracy | Matches MAP scoring | Different from verbal |
| Terms complete | All conditions stated | Missing terms |
| Professional | Error-free, formatted | Typos, sloppy |

### Pass Criteria
- Agreement matches verbal discussion
- No calculation errors
- Client name/details correct
- Follow-up scheduled

## Common Quality Issues

### IDENTIFY Issues
| Issue | Prevention |
|-------|------------|
| Proceeding with low score | Honor the 5+ threshold |
| Missing pain specifics | Probe until you get examples |
| Skipping decision-maker check | Always ask directly |

### ASSESS Issues
| Issue | Prevention |
|-------|------------|
| Not quantifying impact | Push for numbers (hours/dollars) |
| Skipping budget discussion | Ask even if uncomfortable |
| Weak SRO scoring | Use specific criteria |

### MAP Issues
| Issue | Prevention |
|-------|------------|
| Generic module selection | Tie each to specific finding |
| Calculation errors | Double-check math |
| Undocumented constraints | Ask about budget, timeline, team |
| Thin Context Graph | Populate all sections |

### CHART Issues
| Issue | Prevention |
|-------|------------|
| Price differs from verbal | Use same number |
| Vague scope | List specific deliverables |
| Missing follow-up | Schedule before call ends |

## Quality Metrics

### Track Monthly

| Metric | Target | How to Calculate |
|--------|--------|------------------|
| IDENTIFY accuracy | 80%+ qualified proceed | (Qualified that proceed) / (Total qualified) |
| SRO prediction accuracy | 70%+ | (MAP scheduled that complete) / (MAP scheduled) |
| Pricing accuracy | <10% change | (Final price - Quoted price) / Quoted price |
| Proposal revisions | <5% | Proposals revised / Proposals sent |
| First-pass approval | 90%+ | Proposals accepted without changes / Total |

### Quality Review Findings
Log and review patterns:
- Common scoring errors
- Frequently missed questions
- Repeated documentation gaps
- Pricing miscalculations
